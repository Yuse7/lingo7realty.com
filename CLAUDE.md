# Lingo7 Realty — контекст проекта

> Этот файл Claude Code читает автоматически в начале каждой сессии. Здесь — всё, что нужно, чтобы продолжить работу без исходного чата.

## Что это

**Lingo7 Realty** — лендинг (статический сайт) для подготовки к экзамену на **лицензию риелтора в США** (US real estate license), нацеленный на людей, **для которых английский не родной**. Ключевая идея: взять модель exam-prep (как у PrepAgent) и «прошить» в неё языковой инструментарий приложения **Lingo7** (параллельный перевод, тап-сохранение слов, озвучка, интервальное повторение). Так не-носитель закрывает сразу два барьера — предмет и язык, — чего у обычных prep-сервисов нет.

## Стек и архитектура

- **Astro** (статическая генерация, `output: 'static'`, `format: 'directory'` → URL со слешем: `/about/`). Сделано по образцу соседнего проекта `lingoseven.com`. Интеграция `@astrojs/sitemap` сама генерирует `sitemap-index.xml`. Node >= 22.
- Команды: `npm run dev` (дев-сервер), `npm run build` (в `dist/`), `npm run preview`.
- Структура:
  - `src/pages/` — страницы: `index.astro` (лендинг), `about/contacts/terms/privacy.astro`.
  - `src/layouts/` — `BaseLayout.astro` (лендинг: тёмная тема, `global.css`, Head+Header+Footer+скрипт), `LegalLayout.astro` (светлая тема для about/legal, свои стили инлайн).
  - `src/components/` — общие `Head/Header/Footer` + секции лендинга (`HeroSection`, `TrustStrip`, `Compare`, `Languages`, `Library`, `SpacedRepetition`, `MathSection`, `Audience`, `Reviews`, `Faq`, `FinalCta`, `Modals`).
  - `src/scripts/main.ts` — **весь** клиентский JS лендинга (модалки, sticky-CTA, бургер, переключатель языков, ревилы/табы/аккордеон, оркестратор демо, i18n-свап). Подключён `<script src>` в `BaseLayout`, грузится только на лендинге.
  - `src/styles/global.css` — бывший `styles.css` (унаследован от L7 + блок `Lingo7 Realty additions`). При правках стилей переиспользуй L7-часть, не дублируй.
  - `src/i18n/ru.ts` — словарь `I18N_RU` (русская локаль; импортируется в `main.ts`).
  - `public/` — статика, копируется как есть: фавиконки/иконки, `pics/` (только `green2.png`, `phone-clean.svg`), `screens/*.html` (iframe-демо), `robots.txt`, `CNAME`, `.nojekyll`.
- **Локализация: client-side.** По умолчанию сайт на английском (`data-i18n` + английский текст в разметке). При `?lang=ru` `main.ts` подменяет `innerHTML` по словарю. Переведён только русский (язык-шаблон); переключатель в шапке перезагружает с `?lang=<code>`, для прочих кодов остаётся английский. Демо-экраны получают `?lang` в src. Полноценный route-based i18n (как 49 локалей в lingoseven) НЕ делали — нет переводов.

### Интерактивное демо в hero (НЕ сломать)
Внутри телефона в hero — самодостаточный IIFE (последний `<script>`, «Interactive exam-card demo»). Цикл на `async/await`:
1. Экран-каталог `#readerCatalog` (`.examhome`) → палец тапает «Начать обучение» (`#studyStart`).
2. Двуязычная «читалка» (`.reader__pair` = англ. строка `.reader__en[data-line]` + рус. `.reader__ru`): караоке-озвучка по словам, затем палец тапает термин (`lien` в строке 0, `Escrow` в строке 2) → попап → «Перевод» (карточка `#readerCard`) → «Добавить» → тост + новая карточка в деку (`#vocabNew`, `#vocabNew2`).
Данные терминов — объекты `lienData`/`escrowData` в коде. Палец позиционируется через `getBoundingClientRect` (нужен реальный браузер; в headless-скрине анимации не видно).

## Работа с картинками (важно: иначе диалог «протухает»)

Проект visual-heavy: 80+ изображений, многие retina 2500–3024px и айфоновские 1170×2532 (`pics/` ~44 МБ, `inside/` ~9 МБ). Если навалить их в диалог, ловится ошибка `an image in the conversation could not be processed and was removed` — после auto-compact блок картинки бьётся, пересылается каждый ход, и приходится открывать новое окно (теряя контекст). Правила, чтобы этого не было:

- **Не читать тяжёлые оригиналы напрямую.** Лимит API ≈5 МБ и ≤8000px на картинку; рекомендованно ≤1568px по длинной стороне. Перед Read ужимать: `sips -Z 1200 "src.png" --out /tmp/x.png`, читать `/tmp/x.png`. Для `pics/` и `inside/` готовые ужатые копии лежат в `previews/` (зеркало путей, ≤1200px) — смотри их, а не оригиналы. Пересобрать превью: `./make-previews.sh`.
- **Не вываливать пачку скринов за раз.** 1–3 за ход; если нужно сравнить много — точечно и через `previews/`.
- **Скрины вёрстки — только умеренные** (`--window-size=1400,1050`, см. выше). Tall-screenshot во всю страницу не делать — он и бесполезен (см. PDF-заметку), и тяжёлый.
- **Когда ошибка с картинкой всё же появилась** — не открывать сразу новое окно. Сначала `/compact`: он сворачивает историю в текст и выбрасывает сырые картинки, тред сохраняется. Новый чат и так ре-онбордится этим `CLAUDE.md` + папкой `memory/` — заново «погружать» вручную не нужно.
- `green3.png` (6.5 МБ, > лимита API) удалён как неиспользуемый. Не возвращать большие неиспользуемые картинки в корень.
